# Почему это работает без конфликтов?
# Если коротко: оно работает без конфликтов потому что мы разделяем
# "запись информации" и "её применение во времени".

# 1. В чём риск конфликта?
# Если делать вот так:
# нашли 0 - сразу занулили строку и столбец,
# то возникает проблема:
# мы начинаем портить данные, на основе которых ещё
# не закончили обработку. Это создаёт "цепную реакцию".
# - Новый 0 появился из-за нашего действия.
# - Он начинает обнулять всё вокруг.
# - Итог: вся матрица может стать нулевой.

# 2. Что мы делаем на самом деле?
# Мы разделяем процесс на 2 фазы:

# Фаза 1: только пометки.
# Мы частично используем матрицу как хранилище, но аккуратно.
# matrix[i][0] = 0
# matrix[0][j] = 0
# Мы не применяем итоговое обнуление, а просто используем матрицу
# как структуру для хранения меток.
# Здесь мы просто ставим флажки:
# Строка i должна быть обнулена.
# Столбец j должен быть обнулён.
# Важно!
# Мы используем уже существующую структуру (первую строку/столбец),
# но не применяем изменения к остальной матрице.

# Фаза 2: применение этих меток.
# if matrix[i][0] == 0 or matrix[0][j] == 0:
#   matrix[i][j] = 0
# Теперь мы уже не думаем, где были оригинальные нули,
# мы просто читаем готовые инструкции:
# - "эта строка плохая?"
# - "это столбец плохой?"

# 3. Почему нет конфликта?
# Потому что:
# A) Мы используем "текущие нули" только как метки.

# Б) Первая строка и столбец отделены логически.
# Мы специально вынесли их в:
# first_row_zero
# first_col_zero
# Они участвуют как хранилище, но их исходное состояние
# сохраняется отдельно через флаги.

# В) Мы не делаем каскадное обновление.
# Мы гарантируем, что все решения принимаются на основе
# исходного состояния матрицы, а не изменённого в процессе.
# Главное:
# [X] плохой подход:
# - изменил матрицу,
# - потом считал её,
# - и снова изменил её по новым данным.
# [V] наш подход, правильный:
# - сначала "записал план",
# - потом "исполнил план".

# 4. Интуиция.
# Алгоритм не конфликтует, потому что мы чётко отделяем
# "обнаружение нулей" от "их применения".
