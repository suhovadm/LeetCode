Принцип работы (жадный алгоритм):

1. Разбиваем путь на "сегменты" - участки, которые можно пройти за одинаковое количество прыжков.
2. Внутри каждого сегмента отслеживаем, как далеко можно уйти, прыгнув из любой позиции этого сегмента.
3. Когда достигаем конца текущего сегмента ( i == end ), совершаем прыжок в самую дальнюю точку из этого сегмента.

Пример работы.
Для nums = [2, 3, 1, 1, 4]:

* Итерация i = 0: farthest = max(0, 0+2) = 2, i == end ( 0 == 0 ) - jumps = 1, end = 2
* Итерация i = 1: farthest = max(2, 1+3) = 4, i != end ( 1! = 2 )
* Итерация i = 2: farthest = max(4, 2+1) = 4, i == end ( 2 == 2 ) - jumps = 2, end = 4

Цикл завершается, возвращаем jumps = 2

Замечание!
Цикл идёт до len(nums) - 1, поэтому последний элемент не обрабатывается. Если бы мы дошли до
последнего индекса, дополнительный прыжок бы не потребовался.
